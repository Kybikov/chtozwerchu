# ТЗ — «Хто зверху?» Online

Повноцінна онлайн пати-гра у форматі шоу «Хто зверху?»: дві команди
(рожеві «Дівчата» vs сині «Хлопці») грають серію різнотипних раундів із
наскрізним рахунком і фіналом.

## 1. Стек

| Шар | Технологія |
|---|---|
| Realtime-бекенд | **Go 1.24**, `net/http` + `gorilla/websocket` |
| Жива стейт-кімнат | **Redis** (стейт + pub/sub для горизонтального масштабування) |
| Персистентні дані | **PostgreSQL** (`pgx/v5`): каталог пісень/питань, юзери, історія матчів, статистика |
| Фронт | статичний SPA (HTML/CSS/JS), дизайн наявний, транспорт — WebSocket |
| Деплой | **docker-compose** (postgres + redis + app), готово під Coolify/single-host |

## 2. Ролі та ведучий (гібрид)

- **Гість** — заходить лише за іменем + кодом кімнати.
- **Зареєстрований** — email/пароль; йому пишеться історія матчів і статистика.
- **Хост кімнати** — гравець, що створив кімнату (secret `hostToken`); має адмін-контроль.
- **Ведення (гібрид):**
  - авто-раунди — сервер сам веде таймери/підрахунок (музика, вікторина, правда/брехня);
  - субʼєктивні раунди — результат підтверджує хост або голосування команди-суперника (аліас, крокодил).

## 3. Раунди v1

| # | Раунд | Тип | Механіка | Скоринг |
|---|---|---|---|---|
| 1 | **5 слів** | музика | 5 закритих слів фрази, відкриваєш по черзі; команда вгадує пісню | `5 − відкриті` балів (мін. 1 відкрите щоб відповідати) |
| 2 | **Вгадай мелодію** | музика | грає прев'ю-сніпет; хто перший назве пісню/виконавця | спадні бали за часом; buzzer |
| 3 | **Аліас** | слова | гравець пояснює слово не називаючи; команда вгадує на таймер | +1 за кожне вгадане слово за раунд |
| 4 | **Крокодил** | креатив | загадка емодзі-ребусом/описом; команда вгадує | фікс. бали, підтвердж. хостом |
| 5 | **Правда чи брехня** | вікторина | твердження — команда голосує правда/брехня | +бал за правильний колективний голос |

Кожен раунд — плагін, що реалізує інтерфейс `Round` (init / reducer / redacted view).
Нові раунди додаються без зміни ядра.

## 4. Ігровий цикл

```
lobby → (deck із N раундів, типи чергуються) → playing → result → … → final
```

- Хост у лоббі обирає: назви команд, к-ть раундів, набір типів раундів, плейлист/теми.
- Активна команда чергується щораунду.
- Наскрізний рахунок; фінал — переможець + розбивка балів.

## 5. WebSocket-протокол

Клієнт↔сервер, JSON-повідомлення `{ "type": "...", "payload": {...} }`.

- **C→S:** `join`, `create_room`, `leave`, `start_game`, `round_action` (узагальнена дія раунду), `chat`, `set_config`.
- **S→C:** `room_state` (redacted під глядача), `error`, `event` (звук/анімація), `you_are` (ідентифікація).
- Стейт розсилається по кімнатній кімнаті (Redis pub/sub) на всі підключення.
- Приховані дані (загадане слово, закриті слова) НЕ шлються командам, яким не можна їх бачити.

## 6. Дані (Postgres, головні таблиці)

- `users(id, email, pass_hash, display_name, created_at)`
- `songs(id, title, artist, pack, era, aliases[], phrases jsonb, source jsonb)` — сід з `custom-songs.json`
- `questions(id, kind, payload jsonb, pack)` — правда/брехня, аліас-слова, крокодил-загадки
- `matches(id, code, config jsonb, started_at, ended_at, winner)`
- `match_players(match_id, user_id, name, team, score)`
- Redis: `room:{code}` (стейт JSON), `room:{code}:chan` (pub/sub), TTL на неактивні кімнати.

## 7. Етапи реалізації

1. **Фундамент**: Go-модуль, config, WS-хаб, ігрове ядро (state machine), Postgres+Redis стори, docker-compose, міграції, сід каталогу. Раунд «5 слів» end-to-end.
2. **Фронт на WS**: переписати `app.js` з полінгу на WS, зберегти дизайн, екрани лоббі/гри/результату.
3. **Раунди 2-5**: мелодія, аліас, крокодил, правда/брехня + їх екрани.
4. **Акаунти**: guest + реєстрація, історія матчів, статистика.
5. **Поліш**: анімації, звук, реконект, антиспам, деплой.

## 8. Нефункціональні вимоги

- Реконект гравця в активну кімнату (по `playerToken`).
- Кімнати переживають рестарт інстансу (стейт у Redis).
- Кілька інстансів за балансувальником (Redis pub/sub).
- Health: `/health`, `/api/health`.
